# Mail Client Adapter

## Overview
HTTP client adapter implementing the mail client interface for remote service access. This package bridges local client code and the remote Mail Client Service.
Instead of interacting with Gmail directly, the adapter uses an auto-generated HTTP client (from the service’s OpenAPI spec) to send and receive mail data via REST calls.

This ensures:
- Consistent method names and data structures across all `Client` types.
- No changes to user code — it can call Client methods exactly as before.
- Clean separation between business logic (client-side) and transport (HTTP).

### Components

## 1. Imports and Dependencies
```python
from collections.abc import Iterator

from mail_client_api import Client, Message

# Imports from the generated service client
from src.mail_client_service_client.mail_client_api_service_client.client import (

from src.mail_client_service_client.mail_client_api_service_client.api.default import (
    get_messages_summary_messages_get,
    get_message_detail_messages_message_id_get,
    delete_message_messages_message_id_delete,
    mark_message_as_read_messages_message_id_mark_as_read_post,
)
from src.mail_client_service_client.mail_client_api_service_client.models import (
    MessageDetail,
    MessageSummary,

```

Explanation:
- `Iterator` – used to yield messages one at a time for memory-efficient iteration.
- `Message` – the abstract message contract from `mail_client_api`.
- `Client` – the interface (contract) that defines how a mail client should behave.
- `AutoGeneratedClient` – the HTTP client automatically generated from the FastAPI service’s OpenAPI schema; it handles actual HTTP requests and responses.
- `MessageDetail` / `MessageSummary` – Pydantic models representing detailed and summarized email data returned by the service.

## 2. `ServiceMessage` Class
```python
class ServiceMessage(BaseMessage):
    def __init__(self, detail: MessageDetail | MessageSummary):
        self._detail = detail
```
Purpose:
`ServiceMessage` acts as a wrapper around the service’s `MessageDetail` or `MessageSummary` objects, exposing them through the same property interface as the local Message class.
This ensures that calling code can interact with service-returned messages as if they were local objects.

Properties:
Property,	Type,	Description

- `id`, `str`, Unique message identifier. Defaults to empty string if not found
- `from_`, `str`, Sender’s email address. Uses `from_` instead of `from` (Python keyword).
- `to`, `str`, Recipient’s email address.
- `date`, `str`, Message timestamp, typically in ISO-8601 format.
- `subject`, `str`, Message subject line.
- `body`, `str`, Full message body content (only present in `MessageDetail`).

Implementation Detail:
Each property calls `getattr(self._detail, field_name, ""`) to avoid attribute errors when a field is missing.

Example:
```python
msg = ServiceMessage(detail)
print(msg.from_, msg.subject)
```
## 3. `ServiceClientAdapter` Class
```python
class ServiceClientAdapter(Client):
    def __init__(self, base_url: str):
        self._client = AutoGeneratedClient(base_url=base_url)
```
Purpose:
Implements the `Client` interface by delegating method calls to the remote service via the auto-generated HTTP client.
It mirrors the same methods (`get_messages`, `get_message`, `delete_message`, `mark_as_read`) from the local `GmailClient`.

Constructor:
Parameter, Type, Description
`base_url`, `str`, Base URL of the running Mail Client Service (e.g., `"http://localhost:8000"`).

Creates an instance of `AutoGeneratedClient` configured to communicate with that URL.
## Method: `get_messages`
```python
def get_messages(self, max_results: int = 10) -> Iterator[Message]:
    # The generated client exposes a module function for this endpoint.
    resp = get_messages_summary_messages_get.sync(client=self._client, max_results=max_results)
    # The generated client returns a list of MessageSummary objects for this endpoint
    if resp and isinstance(resp, list):
        for msg_summary in resp:
            yield ServiceMessage(msg_summary)
```
Purpose:
Fetches a list of message summaries from the remote service.

Parameters:
`max_results` (`int`, optional): Maximum number of messages to return. Default: `10`.

Returns:
An iterator (`Iterator[Message]`) of `ServiceMessage` objects.
Each object contains summarized information (id, from_, subject, date).

Behavior:
- Calls the service endpoint `GET /messages?max_results={n}`.
- Wraps each returned `MessageSummary` in a `ServiceMessage` and yields it one by one.

Example:
```python
for msg in client.get_messages(max_results=5):
    print(msg.id, msg.subject)
```
## Method: `get_message`
```python
def get_message(self, message_id: str) -> Message:
    resp = self._client.messages.get_message.sync(message_id=message_id)
    return ServiceMessage(resp)
```
Purpose:
Retrieves full details of a single message.

Parameters:
`message_id` (`str`): The unique identifier of the message.

Returns:
A `ServiceMessage` containing the full message details (`id`, `from_`, `to`, `date`, `subject`, `body`).

Behavior:
- Calls the service endpoint `GET /messages/{message_id`}.
- Wraps the returned `MessageDetail` object into a `ServiceMessage`.

Raises:
Any exceptions propagated by the underlying client if the service fails or the message doesn’t exist.

Example:
```python
msg = client.get_message("abc123")
print(msg.subject, msg.body)
```
## Method: `delete_message`
```python
def delete_message(self, message_id: str) -> bool:
    try:
        self._client.messages.delete_message.sync(message_id=message_id)
        return True
    except Exception:
        return False
```
Purpose:
Deletes a message by ID through the service.

Parameters:
`message_id` (`str`): The unique identifier of the message.

Returns:
`True` if the deletion succeeds, otherwise `False`.

Behavior:
- Calls the service endpoint `DELETE /messages/{message_id}`.
- Returns a simple boolean instead of raising exceptions to simplify client handling.

Example:
```python
success = client.delete_message("abc123")
if success:
    print("Message deleted.")
```
## Method: `mark_as_read`
```python
def mark_as_read(self, message_id: str) -> bool:
    try:
        self._client.messages.mark_as_read.sync(message_id=message_id)
        return True
    except Exception:
        return False
```
Purpose:
Marks a message as read.

Parameters:
`message_id` (`str`): The unique identifier of the message.

Returns:
`True` if the operation succeeds, otherwise `False`.

Behavior:
- Calls the service endpoint `POST /messages/{message_id}/mark-as-read` (implemented as `.mark_as_read.sync()` in the generated client).
- Wraps the call in a `try/except` block for safe boolean returns.

Example:
```python
ok = client.mark_as_read("abc123")
print("Marked as read:", ok)
```
## Quick Start

1. **Start the mail client service**:
   ```bash
   uv run uvicorn src.mail_client_service.main:app --reload
   ```

2. **Use the adapter**:
   ```python
   from mail_client_adapter import ServiceClientAdapter
   
   # Connect to the running service
   client = ServiceClientAdapter("http://localhost:8000")
   
   # Use exactly like any other Client implementation
   messages = list(client.get_messages(max_results=5))
   for msg in messages:
       print(f"{msg.from_}: {msg.subject}")
   
   # Get full message details
   if messages:
       full_msg = client.get_message(messages[0].id)
       print(f"Body: {full_msg.body}")
       
       # Mark as read and delete
       client.mark_as_read(messages[0].id)
       client.delete_message(messages[0].id)
   ```

3. **Testing**:
   ```bash
   # Test the adapter (requires running service)
   uv run pytest src/mail_client_adapter/tests/ -v
   ```

## Summary of Function
- `ServiceMessage` standardizes data returned by the Mail Client Service into the same structure as the original local client.
- `ServiceClientAdapter` translates local client method calls into HTTP requests handled by the auto-generated FastAPI client.
- All operations return consistent, predictable types — either `ServiceMessage` objects or boolean success values.
- This adapter completes the "third bridge" in the system architecture, enabling distributed access to the mail client service without breaking the client interface.
